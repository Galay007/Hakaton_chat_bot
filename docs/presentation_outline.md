## Структура финальной презентации (10–15 слайдов)

1. **Обложка** — название проекта, команда, контакты.
2. **Проблема** — зачем администраторам чатов нужна быстрая выгрузка участников.
3. **Целевая аудитория** — владельцы чатов, маркетологи, исследователи.
4. **Решение** — краткое описание бота и ключевых преимуществ (безопасность, скорость, простота).
5. **Демо сценария** — диаграмма пользовательского пути `/start -> отправка файлов -> /export`.
6. **Функциональные требования** — чек-лист ФТ-01...ФТ-09 с отметкой о реализации.
7. **Нефункциональные требования** — конфиденциальность, Docker, документация.
8. **Архитектура** — схема модулей и поток данных (FileManager, DataParser, ExcelGenerator).
9. **Технологический стек** — Python + PTB, Pandas, Docker, BeautifulSoup.
10. **Пример отчёта** — скрин Excel и/или список в чате (использовать `artifacts/sample_output.xlsx`).
11. **Производственная готовность** — деплой через Docker Compose, GitHub-репозиторий.
12. **Планы развития** — поддержка ZIP, Redis для сессий, дополнительные форматы (CSV).
13. **Роли и вклад** — распределение командных обязанностей:
    - Product/Аналитик: сбор требований, UX сценарии, презентация.
    - Backend Lead: архитектура, безопасность, интеграция Telegram API.
    - Data Engineer: парсинг экспорта, дедупликация, Excel.
    - DevOps: Docker, CI/CD, мониторинг.
14. **Риски и их закрытие** — ограничения Telegram, большие файлы, качество данных.
15. **Call to Action** — как подключиться к пилоту или расширить проект.
